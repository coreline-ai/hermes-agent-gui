# Phase 15c Implementation — Messaging Frontend + Profile Archive

Date: 2026-05-26
Status: ✅ Implemented + verified

## Implemented

- `apps/server/api/profile_archive.py`
  - Profile list/clone/export/import routes.
  - tar.gz archive with `MANIFEST.json` and SHA-256 checksums.
  - `ARCHIVE_EXCLUDE_PATTERNS` for device-specific secrets, passkeys, locks, logs, WAL/SHM files.
  - Safe tar import: rejects path traversal, absolute paths, symlinks/hardlinks/device files, checksum mismatch.
  - Import rotates device secret and returns `relogin_required`.
- Messaging UI
  - `/messaging` route with 16 platform card grid.
  - Right-side config drawer with credential form, behavior editor, QR delegated hint, status badge.
  - Save/Test/Disable and Webhook URL rotation actions.
- Profile UI
  - `/profiles` route with profile list, clone, export download, import dropzone.
  - Import success toast and auto logout for re-login flow.
- i18n/a11y
  - en/ko platform label/help keys for all 16 platforms plus common messaging/profile keys.
  - aria labels and keyboard-operable import dropzone/platform buttons.
